About KBase-UI-PY

Welcome to the KBase UI in Python!

This is something of a thought experiment turned to life - what if kbase-ui were implemented as a (mostly) server-side app?

Django top level functionality is organized into apps, which closely align with kbase-ui plugins.

Some plugins have been partially rewritten as apps, initially to get a feel for the process, to identify challenges, to come up with solutions, to build out common functionality.

Features

  • A django web app
  • Uses bootstrap and font-awesome
  • Develop in container, proxy to kbase services

Advantages

  • Python - common language across KBase
  • Server side service comm - faster, more reliable
  • Scalable - don't need to ship entire web app to each browser
  • Widely used framework - Django
  • Global caching strategies - affect all users, no per-user cache warmup

Challenges

  • Keep interactive!
  • Auto-updating ui
  • Shared caches

Status

  • Basic login / logout
  • Building first layer of existing web apps (plugins)
